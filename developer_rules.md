# Developer and AI Agent Guidelines

Please follow these guidelines strictly when working in this repository:

1. **Unit Testing Requirement**:
   - Whenever you implement a new feature or modify existing logic, check if corresponding unit tests are available.
   - If unit tests are missing or incomplete for the modified/added components, implement them.

2. **Test Verification**:
   - Always run all unit tests to verify that changes have not broken existing functionality and that the app builds and works perfectly.

3. **Git Commit Policy**:
   - Add descriptive comments and commit your changes to Git after every change you make (unless explicitly instructed otherwise).

4. **Git Push Policy**:
   - **Do NOT** push your commits to the remote repository (`git push`) unless the user explicitly asks you to do so.
